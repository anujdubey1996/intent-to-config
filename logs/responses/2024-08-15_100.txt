Intent: Define a test setup for evaluating deployment performance and pod startup latency. This includes phases for creating deployments, measuring pod startup latency, and managing pod lifecycle.

Configuration:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: performance-test
spec:
  replicas: 5
  strategy:
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  selector:
    matchLabels:
      app: performance-test
  template:
    metadata:
      labels:
        app: performance-test
    spec:
      containers:
      - name: performance-test
        image: nginx:alpine
        ports:
        - containerPort: 80
        volumeMounts:
        - mountPath: /test-data
          name: test-data-volume
        readinessProbe:
          httpGet:
            path: /healthz
            port: 80
        startupProbe:
          httpGet:
            path: /healthz
            port: 80
          initialDelaySeconds: 10
      volumes:
      - name: test-data-volume
        emptyDir: {}

The provided configuration is a Kubernetes Deployment for testing performance and evaluating pod startup latency. It creates a Deployment with 5 replicas, running an Nginx container from the alpine image. The containers listen on port 80, mount a volume for persistent data storage, and set up both readiness and startup probes. The readiness probe is used to ensure that the container is ready to accept traffic, and the startup probe is used to measure pod startup latency. The emptydir volume is for storing test data persistently between pod restarts. The Deployment strategy used is a RollingUpdate, and the Deployment can only progress if one new pod is being started and no old pod is being terminated at any given time.
