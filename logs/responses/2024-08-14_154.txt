
Create a Kubernetes deployment workload configuration file based on the provided intent.

Intent: Deploy a single replica of the "secondary-container" application using container image from Azure Container Registry with a revision history limit of 3 to retain a history of up to three previous versions.

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: secondary-container-deployment
  labels:
    app: secondary-container
spec:
  replicas: 1
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 0
      maxSurge: 1
  selector:
    matchLabels:
      app: secondary-container
  template:
    metadata:
      labels:
        app: secondary-container
    spec:
      containers:
      - name: secondary-container
        image: <ACR_NAME>.azurecr.io/secondary-container:latest
        imagePullPolicy: Always
        readinessProbe:
          httpGet:
            path: /healthz
            port: 8081
          initialDelaySeconds: 5
          periodSeconds: 10
        livenessProbe:
          httpGet:
            path: /healthz
            port: 8081
          initialDelaySeconds: 15
          periodSeconds: 20
      imagePolicy:
        automaticUpdate:
          tagUpdatePolicy: IfNotPresent
          maximumNumberOfUniqueTags: 3
      volumes:
      - name: logs
        configMap:
          name: secondary-container-logs
      terminationGracePeriodSeconds: 10
      containers:
      - name: secondary-container
        image: <ACR_NAME>.azurecr.io/secondary-container:latest
        imagePullPolicy: Always
        volumeMounts:
        - name: logs
          mountPath: /var/log
          subPath: logs
```

Replace the `<ACR_NAME>` placeholder in the provided YAML file with the actual name of your Azure Container Registry.
