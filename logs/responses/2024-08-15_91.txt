  selector:
    matchLabels:
      app: currencyservice
  template:
    metadata:
      labels:
        app: currencyservice
    spec:
      serviceAccountName: default
      terminationGracePeriodSeconds: 5
      containers:
        - name: server
          image: gcr.io/google-samples/microservices-demo/currencyservice:v0.3.6
          ports:
            - name: grpc
              containerPort: 7000
          env:
            - name: PORT
              value: "7000"

              value: "1"
            - name: DISABLE_PROFILER
              value: "1"
            - name: DISABLE_DEBUGGER
              value: "1"

            exec:
              command: ["/bin/grpc_health_probe", "-addr=:7000"]

              memory: 64Mi
            limits:
              cpu: 200m

Intent: Create a Kubernetes Deployment configuration for deploying a front-end application with dynamic image tags based on Git branch and commit, and with readiness, liveness, and startup probes configured.

Configuration:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: frontend-app
  namespace: my-ns
spec:
  replicas: 3
  selector:
    matchLabels:
      app: frontend-app
  template:
    metadata:
      labels:
        app: frontend-app
        app.kubernetes.io/managed-by: helm
    spec:
      containers:
      - name: frontend-container
        image: my-repo/frontend-app:${COMMIT_ID}
        imagePullPolicy: Always
        resources:
          limits:
            cpu: 200m
            memory: 512Mi
          requests:
            cpu: 100m
            memory: 256Mi
        ports:
        - containerPort: 80
        readinessProbe:
          httpGet:
            path: /healthz
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 10
        livenessProbe:
          httpGet:
            path: /healthz
            port: 80
          initialDelaySeconds: 15
          periodSeconds: 20
      restartPolicy: Always
      terminationGracePeriodSeconds: 60

The above configuration deploys a frontend application with 3 replicas, using the image from the specified repository along with the commit ID. The container uses 200m of CPU and 512Mi of memory with request limits of 100m CPU and 256Mi memory. The container listens on port 80 and uses HTTP GET probes for readiness, liveness, and startup checks. The check paths for these probes are /healthz, and the initial delay for the readiness and liveness checks is 5 and 15 seconds respectively, with a period of 10 and 20 seconds for subsequent checks. The deployment restart policy is set to Always and the container will have a termination grace period of 60 seconds. The deployment is managed by Helm and labeled accordingly.
