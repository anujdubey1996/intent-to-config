Intent: Create and manage a deployment for the CockroachDB UI (roach-ui) service in a development environment.

1. Set the desired number of replicas to 3.
2. Use resources requests and limits to specify resource usage for the containers.
3. Expose the RoachUI service on port 8080.
4. Set up environment variables using secrets for CockroachDB admin credentials.
5. Ensure that the deployment's containers are healthy by using readiness and liveness probes.

Here's the generated Kubernetes deployment configuration:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: cockroachdb-ui
  namespace: development
  labels:
    app: cockroachdb-ui
spec:
  replicas: 3
  selector:
    matchLabels:
      app: cockroachdb-ui
  template:
    metadata:
      labels:
        app: cockroachdb-ui
    spec:
      serviceAccountName: cockroachdb-ui-sa
      containers:
        - name: cockroachdb-ui
          image: cockroachdb/roach-ui:v1.0.0
          ports:
          - name: ui
            containerPort: 8080
          envFrom:
          - secretRef:
              name: cockroachdb-ui-credentials
          resources:
            requests:
              cpu: 100m
              memory: 50Mi
            limits:
              cpu: 200m
              memory: 100Mi
          readinessProbe:
            httpGet:
              path: /healthz
              port: ui
            initialDelaySeconds: 5
            periodSeconds: 10
          livenessProbe:
            httpGet:
              path: /healthz
              port: ui
            initialDelaySeconds: 15
            periodSeconds: 10
```

This Kubernetes deployment configuration creates a Deployment named `cockroachdb-ui` in the `development` namespace, sets the desired number of replicas to 3, and employs environmental variables from the secret named `cockroachdb-ui-credentials`. Additionally, it exposes the RoachUI service on port 8080 through the container port `ui`, and uses readiness and liveness probes to ensure the containers are healthy.
