Intent: Deploy the auth-server on Kubernetes, ensuring that the configuration is templated for flexible deployment based on the Helm values provided.

Configuration:

First, create the following template file auth-server-template.yml:

```yaml
apiVersion:apps/v1
kind: Template
metadata:
  name: authserver-chart-yml
  labels:
    app: auth-server
spec:
  template:
    metadata:
      labels:
        app: auth-server
        chart: {{ .Chart.Name }}
    spec:
      replicas: {{ .Values.replicaCount }}
      selector:
        matchLabels:
          app: auth-server
      serviceAccountName: auth-server
      containers:
      - name: auth-server
        image: auth-server:{{ .Values.imageTag }}
        env:
        {{- with .Values.appSettings }}
          - name: CONFIG_URL
            value: {{ .url }}
        {{- end }}
        ports:
        {{- $ports := .Values.service.ports }}
        {{ range $port_index, $port := $ports }}
        - name: {{ $port.name }}
          containerPort: {{ $port.containerPort }}
          {{- if $port.servicePort }}
          servicePort: {{ $port.servicePort }}
          {{- end }}
          {{- if $port.protocol }}
          protocol: {{ $port.protocol }}
          {{- end }}
        {{- end }}
      livenessProbe:
        httpGet:
          path: /healthz
          port: {{ (index .Values.service.livenessProbe.httpGet.path) | default "http" }}
          scheme: HTTP
        initialDelaySeconds: 15
        periodSeconds: 10
      readinessProbe:
        httpGet:
          path: /ready
          port: {{ (index .Values.service.readinessProbe.httpGet.path) | default "http" }}
          scheme: HTTP
        initialDelaySeconds: 15
        periodSeconds: 10
```

Next, create a values.yaml file for your values:

```yaml
replicaCount: 2
service:
  type: ClusterIP
  livenessProbe:
    httpGet:
      path: /healthz
      port: 80
      scheme: HTTP
  readinessProbe:
    httpGet:
      path: /ready
      port: 80
      scheme: HTTP
appSettings:
  url: http://config-server.example.com
imageTag: latest
```

Finally, you can use Helm to apply these templates and instantiate the auth-server deployment based on your desired settings. For example, if you were deploying the chart in the namespace called my-namespace, run:

```
helm install my-auth-server my-auth-server-chart --namespace my-namespace
```

This template creates a Kubernetes Deployment with the desired configurations based on Helm's values, including the number of replicas, image tag, service type, and probes.
