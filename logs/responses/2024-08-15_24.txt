Intent: Deploy the auth-server with dynamic values from Helm charts, including image configurations, environment settings, resource requests, volume mounts, and annotations. Ensure the deployment has the required security contexts and configurations based on Helm values

For this intent, let's assume the Helm values provided are the following:
- image: auth-server:v3 (the docker image name and tag)
- env.JWT_SECRET: 'my_jwt_secret' (an environment variable for the JWT secret)
- service.type: LoadBalancer (the desired service type)
- podAnnotations.app.kubernetes.io/instance: auth-server (a pod annotation for identifying the instance of the application)
- podSecurityContext.fsGroup: '9999' (a pod security context setting the filesystem group to 9999)
- podSecurityContext.runAsUser: '1000' (a pod security context setting the user id to 1000)
