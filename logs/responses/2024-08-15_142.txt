apiVersion: apps/v1
kind: Deployment

  name: currencyservice

  selector:
    matchLabels:
      app: currencyservice
  template:
    metadata:
      labels:
        app: currencyservice
    spec:
      serviceAccountName: default
      terminationGracePeriodSeconds: 5
      containers:
        - name: server
          image: gcr.io/google-samples/microservices-demo/currencyservice:v0.3.6
          ports:
            - name: grpc
              containerPort: 7000
          env:
            - name: PORT
              value: "7000"

              value: "1"
            - name: DISABLE_PROFILER
              value: "1"
            - name: DISABLE_DEBUGGER
              value: "1"

            exec:
              command: ["/bin/grpc_health_probe", "-addr=:7000"]

              command: ["/bin/grpc_health_probe", "-addr=:7000"]

              memory: 64Mi
            limits:
              cpu: 200m

Intent: Deploy a single replica of a "test-deployment" application, include an initialization container that checks for the availability of a MySQL server before the main application starts. 

```
apiVersion: apps/v1
kind: Deployment
metadata:
  name: test-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: test-deployment
  template:
    metadata:
      labels:
        app: test-deployment
    spec:
      terminationGracePeriodSeconds: 30
      containers:
        - name: main-container
          image: my-test-image
          ports:
            - containerPort: 8080
          env:
            - name: DATABASE_HOST
              value: mongodb
            - name: DATABASE_PORT
              value: 27017
            - name: DATABASE_USERNAME
              value: myuser
            - name: DATABASE_PASSWORD
              value: mypassword
            - name: APPLICATION_PORT
              value: 8080
      initContainers:
        - name: check-db
          image: busybox
          command: ["/bin/sh"]
          args:
            - "-c"
            - "until mysql -h <mysql-server-ip> -u myuser -p mypassword -N -e 'exit' > /dev/null; do :; done"
          env:
            - name: MYSQL_HOST
              value: <mysql-server-ip>
            - name: MYSQL_USER
              value: myuser
            - name: MYSQL_PASSWORD
              value: mypassword
          restartPolicy: OnFailure

```

Replace `<mysql-server-ip>`, `myuser`, and `mypassword` with the IP address, username, and password of the MySQL server respectively. You can modify the DATABASE_HOST, DATABASE_PORT, DATABASE_USERNAME, and DATABASE_PASSWORD in the main-container to match the appropriate MySQL configuration. Make sure that the MySQL server is accessible by the test-deployment pod using its service account.
