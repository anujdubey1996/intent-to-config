Intent: Create a Kubernetes Deployment for a Redis instance with a single replica. The deployment configures resource requests and limits for the Redis container and exposes the default Redis port
Configuration:
apiVersion: apps/v1
kind: Deployment
metadata:
  name: currencyservice
spec:
  selector:
    matchLabels:
      app: currencyservice
  template:
    metadata:
      labels:
        app: currencyservice
    spec:
      serviceAccountName: default
      terminationGracePeriodSeconds: 5
      containers:
        - name: server
          image: gcr.io/google-samples/microservices-demo/currencyservice:v0.3.6
          ports:
            - name: grpc
              containerPort: 7000
          env:
            - name: PORT
              value: "7000"
            - name: DISABLE_TRACING
              value: "1"
            - name: DISABLE_PROFILER
              value: "1"
            - name: DISABLE_DEBUGGER
              value: "1"
          readinessProbe:
            exec:
              command: ["/bin/grpc_health_probe", "-addr=:7000"]
          livenessProbe:
            exec:
              command: ["/bin/grpc_health_probe", "-addr=:7000"]
          resources:
            requests:
              cpu: 50m
              memory: 64Mi
            limits:
              cpu: 200m
              memory: 128Mi

Intent: Deploy a Kubernetes named details-v1 for managing the details application, specifically version v1.
The following environment variables are required for the details-v1 container:
- DB_HOST
- DB_PORT
- DB_DATABASE
- DB_USER
- DB_PASSWORD

The DB_HOST, DB_PORT, DB_DATABASE, and DB_USER environment variables should be sourced from a secret named app-secret-db. The DB_PASSWORD should come from a Kubernetes secret with name app-secret-db-password.
The deployment should have a single replica.
The container should be based on image: details/v1.
The container should expose port: 8080.

Configuration:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: details-v1
spec:
  selector:
    matchLabels:
      app: details-v1
  template:
    metadata:
      labels:
        app: details-v1
    spec:
      containers:
        - name: details-v1-container
          image: details/v1
          ports:
            - containerPort: 8080
          env:
            - name: DB_HOST
              valueFrom:
                secretKeyRef:
                  name: app-secret-db
                  key: db-host
            - name: DB_PORT
              valueFrom:
                secretKeyRef:
                  name: app-secret-db
                  key: db-port
            - name: DB_DATABASE
              valueFrom:
                secretKeyRef:
                  name: app-secret-db
                  key: db-database
            - name: DB_USER
              valueFrom:
                secretKeyRef:
                  name: app-secret-db
                  key: db-user
            - name: DB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: app-secret-db-password
              optional: true
          resources:
            limits:
              memory: "128Mi"
            requests:
              cpu: "500m"
              memory: "64Mi"
      replicas: 1
```
